1 Rencana Pembelajaran Semester Lengkap

Ini adalah Rencana Pembelajaran Semester (RPS) formal untuk mata kuliah EL2007 Sinyal dan Sistem, yang telah direvisi sepenuhnya untuk mengadopsi metodologi “The Sinyal Hunter Initiative” [5]. RPS ini berfungsi sebagai kontrak akademik antara institusi, pengajar, dan mahasiswa, menguraikan tujuan, materi, aktivitas, dan skema penilaian secara transparan [5]. Konsep ini dirancang untuk mengubah mata kuliah Sinyal dan Sistem dari pengalaman belajar pasif menjadi petualangan aktif yang mendorong pemahaman mendalam, keterampilan praktis, kolaborasi, dan refleksi berkelanjutan [6].

1.2 Pendekatan Pembelajaran: The “Sinyal Hunter Initiative”

1.2.1 1. Fondasi Konseptual: Knowledge Map sebagai Peta Pengetahuan

Knowledge Map (Peta Pengetahuan) adalah tulang punggung dari seluruh konsep ini, berfungsi sebagai representasi visual yang dinamis dari seluruh domain Sinyal dan Sistem dan sebagai panduan utama bagi mahasiswa [6, 8]. Peta ini menyoroti interkonektivitas antar konsep, mencegah “silo pengetahuan” [8].

  • Elemen Peta:
    • Titik (Nodes): Merepresentasikan konsep-konsep kunci dalam Sinyal dan Sistem seperti Domain Waktu, Domain Frekuensi, Transformasi Fourier, Transformasi Laplace, Sistem LTI, Konvolusi, Filter, Pole-Zero, dll. [6, 9, 10]. Peta ini distrukturkan ke dalam tiga domain utama: Domain Waktu (DW_), Domain Frekuensi (DF_), dan Domain Kompleks (DK_), serta konsep antar-domain (AD_) seperti Sampling [9, 11, 12].
    • Rute (Edges): Menggambarkan hubungan logis atau matematis antar konsep, menunjukkan bagaimana mereka saling terkait dan bagaimana bergerak dari satu konsep ke konsep lain. Contohnya termasuk “MODELKAN_SBG” (dimodelkan sebagai), “ANALISIS_DLM” (dianalisis dalam), atau “DICIRIKAN_OLEH” (dicirikan oleh) [6, 10, 13].
    • Atribut Node/Edge: Setiap titik atau rute dapat diperkaya dengan detail seperti rumus kunci, grafik representatif, kondisi batas, atau contoh spesifik [6].
  • Tujuan:
    • Membantu mahasiswa melihat gambaran besar dan interkoneksi antar topik [6].
    • Menjadi alat navigasi saat memecahkan masalah dengan mengidentifikasi Titik Mulai (apa yang diketahui dari soal) dan Titik Akhir (solusi yang dicari) [6, 14, 15].

Peta Pengetahuan ini akan dibangun dan diperkaya oleh mahasiswa secara kolaboratif sepanjang semester menggunakan perangkat lunak seperti Quarto dengan Mermaid/Graphviz [16-18].

1.2.2 2. Kendaraan Belajar: Matematika dan Alat Komputasi (Arsenal Sang Pemburu Sinyal)

Untuk menempuh “Rute” dalam Knowledge Map dan mencapai “Titik Akhir”, mahasiswa akan menggunakan berbagai “Kendaraan” [15, 19]. Kendaraan ini secara eksplisit dikaitkan dengan tingkat Taksonomi Bloom yang relevan, menunjukkan kemampuan yang dibutuhkan [15, 19, 20].

  • Kendaraan Matematika (Fundamental) (K_MAT_): Ini adalah bahasa dasar Sinyal dan Sistem, seperti Aljabar, Kalkulus (Diferensial/Integral), Geometri, dan Bilangan Kompleks & Variabel Kompleks. Penguasaan ini setara dengan tingkat Menerapkan (C3) dan Menganalisis (C4) [19, 21-23].
  • Kendaraan Diagram & Visualisasi (K_VIS_): Alat-alat ini menerjemahkan persamaan matematis yang abstrak menjadi representasi visual yang intuitif. Contohnya termasuk Diagram Blok, Plot Sinyal (Domain Waktu, Frekuensi, Kompleks), Pole-Zero Plot, dan Bode Plot. Kendaraan ini mencakup tingkat Memahami (C2) hingga Menciptakan (C6) [19, 21-23].
  • Kendaraan Komputasi (Super Kendaraan) (K_KOM_): Python dengan ekosistem library ilmiahnya (Matplotlib untuk visualisasi, SciPy untuk komputasi numerik sinyal dan sistem, dan SymPy untuk matematika simbolik) dipilih sebagai “Super Kendaraan”. Penggunaan kendaraan ini menargetkan tingkat kognitif tertinggi: Menerapkan (C3), Menganalisis (C4), dan Menciptakan (C6) [19, 21-23].

Pendekatan ini secara fundamental mengubah peran komputasi dalam kurikulum, menjadikannya kendaraan integral untuk eksplorasi, bukan hanya topik tambahan [23, 24].

1.2.3 3. Asesmen Melalui Gamifikasi: “Treasure Hunt”

Asesmen akan diimplementasikan melalui “Treasure Hunt” yang interaktif, mendorong mahasiswa untuk menavigasi Knowledge Map dan memecahkan masalah [25, 26].

  • Mekanisme Harta Karun: Setiap tantangan yang berhasil diselesaikan akan memberikan “kunci” yang membuka “locker” berisi berbagai jenis “harta karun” dengan bobot poin berbeda [25, 27]:
    • Uang (Poin Rendah): Pertanyaan Mengingat (C1) dan Memahami (C2) (definisi, identifikasi konsep) [25, 27].
    • Berlian (Poin Sedang): Pertanyaan Menerapkan (C3) dan Menganalisis (C4) (perhitungan, interpretasi) [25, 27].
    • Emas (Poin Tinggi): Tugas Menciptakan (C6) (desain, realisasi kode Python) [25, 27].
    • Platinum (Poin Sangat Tinggi): Tugas Mengevaluasi (C5) dan Menganalisis Tingkat Lanjut (C4) (diagnosis masalah, optimasi desain) [25, 27].
  • Strategi Pemecahan Soal: Mahasiswa diajarkan untuk menganalisis soal ujian sebagai “Titik Mulai” dan “Titik Akhir” dalam Knowledge Map, kemudian memilih “Rute” dan “Kendaraan” (matematika/Python/diagram) yang sesuai untuk menemukan “Solusi” [14, 16].

1.2.4 4. Kolaborasi & Personalisasi

Konsep ini menyeimbangkan dukungan kelompok dengan akuntabilitas individu [16].

  • Pembuatan Knowledge Map (Kolaborasi Kelompok): Mahasiswa dibagi menjadi kelompok-kelompok kecil untuk secara kolaboratif membangun Knowledge Map master mereka menggunakan Quarto (dengan Mermaid/Graphviz) [16].
  • Tantangan “Treasure Hunt” (Personalisasi Individual): Sebagian besar tantangan diselesaikan secara mandiri oleh setiap mahasiswa [16].
  • Tantangan “Misi Tim” (Bonus Kolaboratif): Beberapa tantangan didesain untuk diselesaikan bersama oleh kelompok, memberikan poin bonus [16].
  • Penilaian Kontribusi (Peer Assessment): Mahasiswa melakukan penilaian sejawat terhadap kontribusi anggota kelompoknya, dan setiap mahasiswa menulis jurnal refleksi individu tentang proses kolaborasi mereka [16].

1.2.5 5. Pembelajaran Berkelanjutan: Jurnal Harian Hunter & Legacy

Untuk memastikan perbaikan kualitas dari tahun ke tahun dan menumbuhkan budaya belajar, konsep ini memperkenalkan mekanisme legacy [28].

  • Jurnal Harian Hunter: Setiap mahasiswa wajib membuat Jurnal Harian Hunter dalam format Quarto (.qmd). Jurnal ini berisi catatan pergumulan, strategi yang dicoba, solusi yang ditemukan, kesalahan yang dibuat, dan pembelajaran baru. Tujuannya adalah untuk mendorong refleksi metakognitif [18, 28].
  • Repositori “Legacy” (GitHub): Di akhir setiap semester, Knowledge Map terbaik, solusi (kode Python & penjelasannya), dan Jurnal Harian (setelah anonimitas) dari mahasiswa tahun tersebut akan disimpan dalam repositori GitHub utama [28, 29]. Ini menjadi “Hall of Fame” digital.
  • Siklus Peningkatan: Mahasiswa baru di tahun berikutnya akan memulai dengan mempelajari karya pendahulu mereka, terutama membaca Jurnal Harian untuk memahami tantangan umum dan strategi efektif. Tugas mereka adalah tidak hanya mereplikasi, tetapi juga meningkatkan Knowledge Map, menemukan “Rute” yang lebih baik, menggunakan “Kendaraan” yang lebih canggih, atau mencapai “Skor” yang lebih tinggi dalam tantangan [28, 30].

1.3 Struktur Teknis: Implementasi dengan GitHub dan Quarto

Seluruh konsep ini dapat diimplementasikan secara efisien menggunakan GitHub sebagai fondasi dan Quarto sebagai alat utama untuk pembuatan konten [31, 32].

  • Struktur Repositori GitHub: Satu repositori utama (Sinyal-Sistem-TE-ITB) dengan sub-folder yang terstruktur rapi: 00_Materi_Ajar/, 01_Tantangan_Treasure_Hunt/, 02_Knowledge_Maps/, 03_Hunter_Journals/, 04_Treasure_Submissions/, dan 05_Legacy_Archives/ [33, 34].
  • Penggunaan Quarto (.qmd): Semua dokumen kunci (Knowledge Map, Jurnal Harian, Solusi Harta Karun, bahkan Materi Ajar) akan dibuat dalam format Quarto (.qmd). Quarto mendukung penulisan teks Markdown, blok kode Python (dengan output dan plot Matplotlib), diagram Mermaid/Graphviz, dan rumus LaTeX, semuanya dalam satu file [18, 31].
  • Alur Kerja GitHub untuk Mahasiswa: Mahasiswa mengikuti alur Fork -> Clone -> Branch -> Edit .qmd -> Commit -> Push -> Pull Request, yang mengajarkan mahasiswa version control dan kolaborasi standar industri [31, 35, 36].
  • GitHub Pages & GitHub Actions: Digunakan untuk secara otomatis merender file .qmd menjadi HTML setiap kali ada perubahan dan menerbitkannya ke GitHub Pages, menciptakan situs web dinamis dari seluruh ekosistem belajar. GitHub Actions juga bisa digunakan untuk auto-grading beberapa tantangan [31, 37].
