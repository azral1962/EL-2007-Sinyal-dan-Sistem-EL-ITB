1. Fondasi Konseptual: Peta Pengetahuan Universal

Peta Pengetahuan (Knowledge Map) adalah tulang punggung dari seluruh konsep ini, berfungsi sebagai representasi visual dinamis dari seluruh domain Sinyal dan Sistem. Peta ini menyoroti interkonektivitas antar konsep, mencegah “silo pengetahuan” yang sering terjadi pada pendekatan silabus linear tradisional.

Elemen Peta:

  • Titik (Nodes): Merepresentasikan konsep-konsep kunci dalam Sinyal dan Sistem (misalnya, Domain Waktu, Domain Frekuensi, Transformasi Fourier). Peta ini distrukturkan ke dalam tiga domain utama: Domain Waktu, Domain Frekuensi, dan Domain Kompleks.

  • Rute (Edges): Menggambarkan hubungan logis atau matematis antar konsep (misalnya, “memiliki representasi di,” “dapat dianalisis dengan”).

  • Atribut Node/Edge: Detail tambahan seperti rumus kunci atau contoh spesifik dapat disertakan.

Tujuan Peta Pengetahuan:

  • Membantu mahasiswa melihat gambaran besar dan interkoneksi antar topik.

  • Menjadi alat navigasi saat memecahkan masalah dengan mengidentifikasi Titik Mulai (informasi yang diketahui dari soal) dan Titik Akhir (solusi yang dicari).

2. Kendaraan Belajar: Arsenal Sang Pemburu Sinyal

Untuk menempuh “Rute” dalam Knowledge Map dan mencapai “Titik Akhir,” mahasiswa akan menggunakan berbagai “Kendaraan”. Kendaraan ini dikaitkan dengan tingkat Taksonomi Bloom yang relevan, menunjukkan kemampuan kognitif yang dibutuhkan.

  • Kendaraan Matematika (Fundamental): Bahasa dasar Sinyal dan Sistem (misalnya, Aljabar, Kalkulus, Bilangan Kompleks). Target Bloom: Menerapkan (C3) dan Menganalisis (C4).
  • Kendaraan Diagram & Visualisasi: Merepresentasikan sistem dan sinyal secara visual (misalnya, Diagram Blok, Plot Sinyal, Pole-Zero Plot, Bode Plot). Target Bloom: Memahami (C2), Menerapkan (C3), Menganalisis (C4), Menciptakan (C6).
  • Kendaraan Komputasi (Super Kendaraan): Alat ampuh seperti Python dengan library Matplotlib (visualisasi), SciPy (komputasi numerik), dan SymPy (matematika simbolik) untuk menerapkan kendaraan matematis dan visualisasi secara efisien. Target Bloom: Menerapkan (C3), Menganalisis (C4), Menciptakan (C6).

Pendekatan ini menjadikan komputasi sebagai kendaraan integral untuk eksplorasi, bukan hanya topik tambahan.

3. Asesmen Melalui Gamifikasi: “Treasure Hunt”

Asesmen diimplementasikan melalui “Treasure Hunt” yang interaktif, mendorong mahasiswa untuk menavigasi Knowledge Map dan memecahkan masalah.

  • Mahasiswa berperan sebagai “Sinyal Hunter” yang berburu “Rumus Universal Sinyal” dengan memecahkan teka-teki terkait materi.

  • Tantangan adalah soal-soal asesmen yang tersebar di setiap “Titik (Node)” dalam Knowledge Map.

  • Setiap tantangan yang berhasil diselesaikan memberikan “kunci” yang membuka “locker” berisi berbagai jenis “harta karun” dengan bobot poin berbeda:

    • *Uang (Poin Rendah): Pertanyaan Mengingat (C1) dan Memahami (C2).

    • Berlian (Poin Sedang): Pertanyaan Menerapkan (C3) dan Menganalisis (C4).

    • Emas (Poin Tinggi): Tugas Menciptakan (C6) (desain, realisasi kode Python).

    • Platinum (Poin Sangat Tinggi): Tugas Mengevaluasi (C5) dan Menganalisis Tingkat Lanjut (C4).

    • Total skor dari harta karun ini akan menjadi nilai akhir mahasiswa.

4. Kolaborasi & Personalisasi

Konsep ini menyeimbangkan dukungan kelompok dengan akuntabilitas individu:

  • Pembuatan Knowledge Map (Kolaborasi Kelompok): Mahasiswa dibagi dalam kelompok untuk membangun Knowledge Map master menggunakan Quarto dengan Mermaid/Graphviz/PlantUML, mendorong diskusi dan pemahaman bersama.

  • Tantangan “Treasure Hunt” (Personalisasi Individual): Sebagian besar tantangan diselesaikan secara mandiri, dengan poin dan kunci milik pribadi.

  • Tantangan “Misi Tim” (Bonus Kolaboratif): Beberapa tantangan didesain untuk diselesaikan bersama kelompok, memberikan poin bonus.

  • Penilaian Kontribusi (Peer Assessment): Mahasiswa melakukan penilaian sejawat untuk memastikan kontribusi yang adil.

5. Pembelajaran Berkelanjutan: Jurnal Harian Hunter & Legacy

Untuk memastikan perbaikan kualitas dari tahun ke tahun dan menumbuhkan budaya belajar, diperkenalkan mekanisme legacy:

  • Jurnal Harian Hunter: Setiap mahasiswa wajib membuat Jurnal Harian Hunter dalam format Quarto (.qmd), berisi catatan pergumulan, strategi, solusi, dan kesalahan untuk mendorong refleksi metakognitif (belajar tentang cara belajar).

  • Repositori “Legacy” (GitHub): Knowledge Map terbaik, solusi, dan Jurnal Harian (setelah anonimitas) dari mahasiswa tahun sebelumnya akan disimpan dalam repositori GitHub utama di folder 05_Legacy_Archives/.

  • Siklus Peningkatan: Mahasiswa baru di tahun berikutnya akan mempelajari karya pendahulu mereka, dan tugas mereka adalah meningkatkan Knowledge Map atau solusi, memastikan kualitas kuliah terus meningkat secara organik.

6. Implementasi Teknis Menggunakan GitHub dan Quarto

Seluruh konsep diimplementasikan secara efisien menggunakan GitHub sebagai fondasi dan Quarto sebagai alat utama untuk pembuatan konten.

  • Struktur Repositori GitHub: Satu repositori utama (Sinyal-Sistem-TE=-LITB) dengan sub-folder terstruktur rapi untuk materi ajar, Knowledge Map kelompok, tantangan Treasure Hunt, solusi mahasiswa, jurnal harian, dan arsip “Legacy”.

  • Penggunaan Quarto (.qmd): Semua dokumen kunci dibuat dalam format Quarto, yang mendukung teks Markdown, blok kode Python, diagram Mermaid/Graphviz/plantUML, dan rumus LaTeX dalam satu file. File ini dapat dirender menjadi HTML (untuk web interaktif di GitHub Pages), PDF, atau format lain.

  • Alur Kerja GitHub untuk Mahasiswa: Mahasiswa mengikuti alur kerja standar industri Fork -> Clone -> Branch -> Edit .qmd -> Commit -> Push -> Pull Request. Ini mengajarkan version control dan kolaborasi standar industri.

  • GitHub Pages & GitHub Actions: Digunakan untuk secara otomatis merender file .qmd menjadi HTML dan menerbitkannya ke GitHub Pages, menciptakan situs web dinamis. GitHub Actions juga bisa digunakan untuk auto-grading beberapa tantangan.

8. Peta Jalan Operasional Mingguan (Contoh Detail untuk Minggu 1-4)

Berikut adalah contoh rincian untuk beberapa minggu awal perkuliahan, yang mengintegrasikan topik inti, fokus peta pengetahuan, aktivitas pembelajaran, aktivasi kendaraan, dan contoh tantangan/harta karun.

  • Minggu 1: Pendahuluan
    • Topik Inti: Konteks, Latar Belakang, Ringkasan Kuliah.
    • Fokus Peta Pengetahuan: Nodes: Sinyal, Sistem, Domain Waktu, Domain Frekuensi. Edges: dimodelkan sebagai, dianalisis dalam.
    • Aktivitas Pembelajaran: Kuliah pembuka “The Sinyal Hunter Initiative,” tur Peta Pengetahuan, pengenalan ekosistem GitHub/Quarto. Asinkron: Setup lingkungan kerja (Git, Python), menonton video Orasi Ilmiah.
    • Aktivasi Kendaraan: GitHub, Quarto, Python.
    • Tugas & Harta Karun: Emas (5 Poin): Kuis: “Sebutkan tiga contoh sinyal dan sistem dalam kehidupan sehari-hari.” Tugas Awal (Wajib): Fork repositori & buat file Jurnal Harian Hunter.
  • Minggu 2: Sinyal
    • Topik Inti: Transformasi Waktu, Periodisitas, Sinyal Primitif.
    • Fokus Peta Pengetahuan: Nodes: Sinyal CT/DT, u(t), δ(t), Sinyal Periodik. Edges: transformasi waktu pada, disusun dari.
    • Aktivitas Pembelajaran: Workshop interaktif tentang transformasi waktu (shifting, scaling, reversal). Asinkron: Latihan menggambar sinyal hasil transformasi. Pengenalan sinyal impuls dan step.
    • Aktivasi Kendaraan: Fundamental: Aljabar. Visualisasi: Plot Sinyal. Komputasi: matplotlib.pyplot.plot untuk memvisualisasikan sinyal.
    • Tugas & Harta Karun: Platinum (20 Poin): “Gambarkan sinyal y(t)=2x(−2t+4) jika x(t) adalah sinyal segitiga.” Uang (40 Poin): “Buat fungsi Python yang menerima sebuah sinyal (array) dan parameter transformasi, lalu mengembalikan sinyal yang telah ditransformasi.”.
  • Minggu 3: Sistem
    • Topik Inti: Properti Sistem, LTI, Respons Impuls.
    • Fokus Peta Pengetahuan: Nodes: Sistem, Linearitas, Time-Invariance, Sistem LTI, Respons Impuls. Edges: memiliki properti, dicirikan oleh.
    • Aktivitas Pembelajaran: Diskusi properti sistem (kausalitas, stabilitas, dll.). Pengenalan konsep LTI dan pentingnya respons impuls. Asinkron: Latihan mengklasifikasikan sistem berdasarkan propertinya.
    • Aktivasi Kendaraan: Fundamental: Aljabar. Visualisasi: Diagram input-output.
    • Tugas & Harta Karun: Emas (10 Poin): Kuis: “Apakah sistem y[n]=nx[n] time-invariant?” Platinum (25 Poin): “Buktikan apakah sistem y(t)=∫−∞t​x(τ)dτ linear dan time-invariant.”.
  • Minggu 4: Konvolusi
    • Topik Inti: Konvolusi CT dan DT.
    • Fokus Peta Pengetahuan: Nodes: Konvolusi. Edges: output dihitung dengan (menghubungkan Input/Respons Impuls ke Output).
    • Aktivitas Pembelajaran: Workshop perhitungan konvolusi grafis. Asinkron: Latihan perhitungan konvolusi manual. Sesi Lab Komputasi: Menggunakan scipy.signal.convolve dan membandingkan hasilnya dengan perhitungan manual.
    • Aktivasi Kendaraan: Fundamental: Kalkulus Integral. Visualisasi: Plot sinyal input, h(t), dan output. Komputasi: scipy.signal.convolve.
    • Tugas & Harta Karun: Platinum (30 Poin): “Hitung y(t)=u(t)∗e−atu(t).” Uang (50 Poin): “Buat script Python untuk memvisualisasikan proses konvolusi grafis secara animasi.” Berlian (75 Poin): “Modelkan efek gema audio sebagai konvolusi. Rancang respons impuls h[n] untuk menciptakan efek gema tiga kali dengan pelemahan. Terapkan pada file audio.wav.”.

Tentu, berikut adalah kelanjutan Rencana Pembelajaran Semester (RPS) mata kuliah EL2007 Sinyal dan Sistem: The “Sinyal Hunter Initiative” mulai dari Minggu 5 hingga selesai, dengan menggabungkan detail operasional dan kerangka akademik yang telah dibahas.

Konsep “Sinyal Hunter Initiative” ini dirancang untuk mengubah pengalaman belajar pasif menjadi petualangan aktif yang mendorong pemahaman mendalam, keterampilan praktis, kolaborasi, dan refleksi berkelanjutan. Penilaian didesain ulang sepenuhnya untuk mendorong keterlibatan berkelanjutan dan penghargaan terhadap berbagai jenis penguasaan materi melalui “Treasure Hunt” atau Perburuan Harta Karun. Total skor harta karun menyumbang 40% dari nilai akhir, ditambah dengan dua UTS (masing-masing 20%) dan UAS (20%). Selain itu, Jurnal Harian Hunter adalah prasyarat kelulusan, dan Misi Tim menawarkan poin bonus kolaboratif.

Berikut adalah RPS mingguan yang terperinci:

Minggu 5: Sistem LCCDE – Representasi & Solusi * Capaian Pembelajaran Mata Kuliah (CPMK): Mampu memodelkan sistem menggunakan LCCDE (Linear Constant-Coefficient Differential Equation) dan merealisasikannya dalam diagram blok. * Bahan Kajian (Topik Inti): Persamaan Diferensial/Diferensial Koefisien Konstan (LCCDE), dan referensi Jam Sesi 7, 8, 9. * Fokus Peta Pengetahuan: * Titik (Nodes): LCCDE, Diagram Blok. * Rute (Edges): Hubungan antar konsep seperti “adalah model untuk” dan “direalisasikan dengan”. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): * Sinkron: Menurunkan LCCDE dari sistem fisik (misal: RLC), dan menggambar diagram blok dari LCCDE. * Asinkron: Latihan mencari respons sistem LCCDE (solusi homogen & partikular). * Simulasi Komputer. * Kendaraan Belajar & Teknologi: * Fundamental: Persamaan Diferensial. * Visualisasi: Diagram Blok. * Komputasi: scipy.signal.lfilter untuk simulasi sistem DT. * Tugas & Harta Karun (Asesmen): * Platinum (30 Poin): “Tentukan respons total dari sistem y[n]−0.5y[n−1]=x[n] untuk input x[n]=(0.8)nu[n].”. * Uang (60 Poin): “Implementasikan sistem LCCDE orde-2 dalam Python dan plot respons step-nya.”. * Bobot Penilaian: 3%.

Minggu 6: Deret Fourier – Eigenfunction & Representasi Sinyal Periodik * CPMK: Mampu merepresentasikan sinyal periodik menggunakan Deret Fourier. * Bahan Kajian (Topik Inti): Eigenfunction, Deret Fourier (CTFS/DTFS), dan referensi Jam Sesi 10, 11, 12. * Fokus Peta Pengetahuan: * Titik (Nodes): Deret Fourier (CTFS/DTFS), Eigenfunction. * Rute (Edges): Hubungan “adalah representasi frekuensi untuk” → Sinyal Periodik. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): * Sinkron: Pengenalan konsep eigenfunction ejωt pada sistem LTI, dan workshop perhitungan koefisien Deret Fourier. * Asinkron: Latihan mencari koefisien Fourier dan menggunakan tabel properti. * Latihan Simbolik. * Kendaraan Belajar & Teknologi: * Fundamental: Aljabar Kompleks, Kalkulus. * Komputasi: SymPy untuk integral simbolik, Matplotlib untuk plot spektrum. * Tugas & Harta Karun (Asesmen): * Platinum (35 Poin): “Tentukan koefisien Deret Fourier dari sinyal gelombang persegi.”. * Uang (60 Poin): “Buat script Python yang mensintesis ulang sinyal persegi menggunakan N suku pertama dari Deret Fourier-nya. Tunjukkan bagaimana Gibbs phenomenon muncul.”. * Bobot Penilaian: 3%.

Minggu 7: Sifat Deret Fourier & Aplikasi Filter * CPMK: Mampu menganalisis respons sistem LTI terhadap input periodik dan memahami konsep filtering. * Bahan Kajian (Topik Inti): Sifat Deret Fourier, Sistem LTI sebagai Filter, dan referensi Jam Sesi 12, 13, 14, 15. * Fokus Peta Pengetahuan: * Titik (Nodes): Respons Frekuensi, Filter. * Rute (Edges): Hubungan “output sistem LTI adalah” → y(t)=∑akH(jkω0)ejkω0t. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): * Sinkron: Diskusi sifat-sifat Deret Fourier, dan konsep filtering sebagai pengubahan magnituda/fasa komponen frekuensi. * Asinkron: Menganalisis output filter sederhana untuk input periodik. * Analisis Sistem Filter Sederhana. * Kendaraan Belajar & Teknologi: * Fundamental: Aljabar Kompleks. * Visualisasi: Plot Spektrum (Magnituda & Fasa). * Tugas & Harta Karun (Asesmen): * Berlian (80 Poin): “Diberikan sebuah sinyal periodik yang terkontaminasi noise frekuensi tinggi. Rancang sebuah sistem LCCDE sederhana (filter low-pass) dan tunjukkan secara analitis dan simulasi bagaimana sistem tersebut membersihkan sinyal.”. * Bobot Penilaian: 3%.

Minggu 8: UJIAN TENGAH SEMESTER 1 * Bahan Kajian (Topik Inti): Materi Minggu 1-7. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): Ujian Daring/Luring. * Tugas & Harta Karun (Asesmen): Soal setara level Emas dan Platinum. * Bobot Penilaian: 20%.

Minggu 9-10: Transformasi Fourier (CTFT & DTFT) & Sistem LCCDE * CPMK: Mampu memodelkan sinyal aperiodik dan sistem LTI menggunakan Transformasi Fourier (CTFT & DTFT). * Bahan Kajian (Topik Inti): Transformasi Fourier CT/DT, Sifat-sifat, Analisis Sistem LCCDE, dan referensi Jam Sesi 16, 17, 18, 19, 20, 21. * Fokus Peta Pengetahuan: * Titik (Nodes): CTFT, DTFT, Respons Frekuensi. * Rute (Edges): Hubungan “adalah representasi frekuensi untuk” → Sinyal Aperiodik, “adalah generalisasi dari” → Deret Fourier, dan “adalah analog waktu-diskrit dari” → CTFT. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): * Sinkron: Menurunkan CTFT dari CTFS, workshop perhitungan CTFT, diskusi sifat dualitas, konvolusi, dll. Menurunkan DTFT, analisis sistem LCCDE di domain frekuensi. * Asinkron: Latihan menggunakan tabel pasangan dan sifat-sifat CTFT, latihan mencari respons frekuensi H(ejω) dari sistem DT. * Kuliah Interaktif, Workshop Sifat-sifat, Sesi Lab Komputasi. * Kendaraan Belajar & Teknologi: * Fundamental: Kalkulus, Deret Geometri. * Komputasi: SymPy untuk transformasi simbolik, scipy.signal.freqz untuk plot respons frekuensi. * Tugas & Harta Karun (Asesmen): * Platinum (30 Poin): “Gunakan sifat konvolusi untuk mencari Transformasi Fourier dari sinyal segitiga.”. * Uang (65 Poin): “Untuk filter moving average, turunkan respons frekuensinya secara analitis, lalu plot magnituda dan fasanya menggunakan Python. Identifikasi frekuensi di mana filter ini menolak sinyal sepenuhnya (nulls).”. * Bobot Penilaian: 5%.

Minggu 11: Filter & Karakterisasi Waktu-Frekuensi * CPMK: Mampu mengkarakterisasi filter berdasarkan respons frekuensi dan memahami trade-off waktu-frekuensi. * Bahan Kajian (Topik Inti): Filter Ideal vs. Non-Ideal, Filter Butterworth, Bode Plot, dan referensi Jam Sesi 22, 23, 24. * Fokus Peta Pengetahuan: * Titik (Nodes): Filter Ideal, Filter Butterworth, Bode Plot. * Rute (Edges): Hubungan “didesain untuk memenuhi” → Spesifikasi Frekuensi. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): * Sinkron: Diskusi trade-off antara idealitas di domain frekuensi dan perilaku di domain waktu (misal: ringing), pengenalan filter Butterworth. * Asinkron: Analisis filter orde rendah. * Studi Kasus, Analisis Komparatif, Sesi Lab Desain Filter. * Kendaraan Belajar & Teknologi: * Visualisasi: Bode Plot. * Komputasi: scipy.signal.butter, scipy.signal.bode. * Tugas & Harta Karun (Asesmen): * Berlian (90 Poin): “Bandingkan respons step dan respons frekuensi dari filter Butterworth orde 2 dan orde 8. Jelaskan trade-off antara ketajaman roll-off di domain frekuensi dan overshoot di domain waktu.”. * Bobot Penilaian: 4%.

Minggu 12: Sampling & Aliasing * CPMK: Mampu menjelaskan fenomena sampling, teorema Nyquist, dan aliasing. * Bahan Kajian (Topik Inti): Representasi Sinyal CT dengan DT, Aliasing, Rekonstruksi, dan referensi Jam Sesi 25, 26, 27. * Fokus Peta Pengetahuan: * Titik (Nodes): Sampling, Teorema Nyquist, Aliasing. * Rute (Edges): Hubungan “menghubungkan” → Sinyal CT dan DT. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): * Sinkron: Demonstrasi visual efek aliasing, penurunan teorema sampling. * Asinkron: Latihan menentukan laju Nyquist, studi kasus: pemrosesan sinyal CT dengan sistem DT. * Simulasi Komputer. * Kendaraan Belajar & Teknologi: * Visualisasi: Plot spektrum sinyal sebelum dan sesudah sampling. * Komputasi: Matplotlib. * Tugas & Harta Karun (Asesmen): * Uang (70 Poin): “Buat script Python yang mensimulasikan sampling sebuah sinyal sinus. Tunjukkan secara visual apa yang terjadi pada sinyal hasil rekonstruksi ketika laju sampling di bawah laju Nyquist (aliasing).”. * Bobot Penilaian: 3%.

Minggu 13: Transformasi Laplace – Definisi, RoC, Sifat * CPMK: Mampu merepresentasikan sinyal menggunakan Transformasi Laplace dan menganalisis Region of Convergence (RoC). * Bahan Kajian (Topik Inti): Transformasi Laplace, RoC, Pole-Zero Plot, dan referensi Jam Sesi 28, 29. * Fokus Peta Pengetahuan: * Titik (Nodes): Transformasi Laplace, Region of Convergence (RoC), Pole-Zero Plot. * Rute (Edges): Hubungan “adalah generalisasi dari” → CTFT. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): * Sinkron: Motivasi Laplace (analisis sinyal tak stabil), konsep RoC dan kaitannya dengan kausalitas/stabilitas. * Asinkron: Latihan mencari Transformasi Laplace dan menentukan RoC. * Kuliah Interaktif, Workshop Analisis RoC. * Kendaraan Belajar & Teknologi: * Fundamental: Variabel Kompleks. * Visualisasi: Pole-Zero Plot di s-plane. * Tugas & Harta Karun (Asesmen): * Platinum (35 Poin): “Tentukan Transformasi Laplace dan RoC untuk sinyal x(t)=e−2tu(t)+e−tu(−t).”. * Bobot Penilaian: 3%.

Minggu 14: Inversi Laplace & Analisis Sistem LTI * CPMK: Mampu melakukan inversi Transformasi Laplace dan menganalisis sistem LTI di domain-s. * Bahan Kajian (Topik Inti): Inversi Laplace, Fraksi Parsial, Fungsi Sistem H(s), Stabilitas, dan referensi Jam Sesi 30, 31, 32, 33. * Fokus Peta Pengetahuan: * Titik (Nodes): Inversi Laplace, Fungsi Sistem H(s). * Rute (Edges): Hubungan “menentukan” → Stabilitas & Respons Impuls. * Bentuk & Metode Pembelajaran (Rute & Aktivitas): * Sinkron: Workshop ekspansi fraksi parsial, analisis sistem LTI menggunakan H(s) dan pole-zero plot. * Asinkron: Latihan inversi Laplace dan menentukan respons sistem. * Analisis Geometris Pole-Zero. * Kendaraan Belajar & Teknologi: * Fundamental: Aljabar (Partial Fractions). * Komputasi: sympy.apart dan sympy.inverse_laplace_transform. * Tugas & Harta Karun (Asesmen): * Berlian (100 Poin): “Sebuah sistem LTI memiliki fungsi sistem H(s)=(s+2)(s−3)s−1​.
